# Politique de sécurité

Écluse est un outil de protection des données. Sa sécurité est une
priorité absolue, et nous accueillons avec gratitude tout signalement
responsable de vulnérabilité.

## Signaler une vulnérabilité

Si vous découvrez une faille de sécurité, merci de **ne pas** ouvrir
d'issue publique. Contactez-nous en privé :

- Email : security@ecluse.dev
- Ou via les « Security advisories » privés de GitHub (onglet Security
  du dépôt → Report a vulnerability).

Merci d'inclure, dans la mesure du possible : une description de la
faille, les étapes pour la reproduire, son impact potentiel, et toute
suggestion de correction.

## Notre engagement

- Accusé de réception sous 72 heures.
- Évaluation et retour sur la recevabilité sous 7 jours.
- Tenue informée de la personne qui signale jusqu'à résolution.
- Crédit public (si vous le souhaitez) une fois la faille corrigée.

Nous demandons en retour un délai raisonnable pour corriger avant toute
divulgation publique (divulgation coordonnée).

## Périmètre

Tout ce qui touche à la confidentialité des données traitées par Écluse
nous intéresse particulièrement : fuite de données vers le LLM,
exposition de la table de pseudonymisation, défaut de chiffrement,
contournement de la détection, fuite via les journaux d'audit.

## Versions supportées

Le projet est en développement actif. Seule la dernière version publiée
reçoit des correctifs de sécurité.
